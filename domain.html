Literature Survey

Current research highlights a growing need for effective energy handling in both residential and industrial sectors. Existing studies (Khan et al., 2018) focus predominantly on smart meters but often lack device-level granularity — they measure a building's total consumption but cannot pinpoint which specific appliance is wasting energy.

Recent advancements in low-cost IoT microcontrollers such as the ESP32 and ESP8266 have opened doors for real-time edge computing, enabling localized signal processing and calibration directly on the hardware (Garcés, 2025). Parallel research in occupancy detection has demonstrated that combining Passive Infrared (PIR) sensors with RCWL microwave radar significantly reduces false-negative readings compared to either sensor used in isolation.

Despite this progress, a fully integrated pipeline — from occupancy detection through to appliance-level energy actioning — remains largely absent from commercially available or research-grade systems.

Research Gap

Core Gap Identified:

Most existing systems provide aggregated building-level energy data but fail to identify which specific appliance is consuming energy in an unoccupied room. There is a clear absence of a system that tightly integrates:

  • Real-time occupancy detection (multi-sensor fusion)
  • Device-level current monitoring (per-appliance metering)
  • Automated, AI-driven corrective actions (not just dashboards)

Research Problem

"How can IoT sensor fusion be used to automatically detect energy waste at the appliance level in unoccupied spaces and provide real-time corrective recommendations?"

Organizations lack a unified system that simultaneously monitors device-level energy consumption and detects room occupancy. Manual intervention is required to turn off appliances in vacant spaces, resulting in unnecessary energy waste, increased operational costs, and an enlarged carbon footprint.

Research Objectives

1
Device-Level Monitoring
Design and implement an energy monitoring module using the ACS712 current sensor interfaced with ESP8266 for wireless transmission via REST API.
2
Occupancy Detection
Develop a reliable occupancy detection subsystem that fuses PIR and RCWL microwave radar sensor data via ESP32 for high-accuracy human presence identification.
3
Real-Time Data Pipeline
Build a data pipeline that correlates occupancy status with per-appliance energy metrics and stores results in a structured database for analysis.
4
AI-Driven Recommendations
Create an intelligent recommendation engine that automatically alerts users or controls devices when energy waste is detected in unoccupied rooms.
5
System Evaluation
Evaluate system accuracy, energy savings achieved, and overall user experience through controlled real-world testing at SLIIT.

Methodology

Energy Monitoring

ACS712 current sensor reads real-time amperage per appliance. Data is processed by an ESP8266 and pushed to the backend via HTTP API over Wi-Fi.

Occupancy Sensing

A dual-sensor module combines PIR (motion) and RCWL microwave radar connected to an ESP32 for reduced false negatives and improved accuracy in all lighting conditions.

Hardware Build

Sensors are housed in custom-built "Step Boxes" powered by independent battery units for modularity. All firmware is written and calibrated using Arduino IDE.

Backend & Database

A Python REST API receives sensor payloads, validates data, and writes to MongoDB — a NoSQL document database ideal for flexible, time-series IoT data. Business logic correlates occupancy events with per-appliance energy readings.

Flutter Mobile App

A cross-platform Flutter mobile application (Android & iOS) provides on-the-go monitoring — real-time energy usage, occupancy alerts, and device control, all communicating with the Python API.

AI Recommendation

When occupancy drops to zero but current draw remains, the system triggers an automated alert or relay command, guiding users or directly cutting power to wasteful appliances.

Dashboard & Analytics

A web-based management dashboard built with HTML5, CSS3, and Bootstrap visualises per-device consumption trends, occupancy history, and estimated savings.
